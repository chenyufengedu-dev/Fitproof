## 一、产品定义

**一句话**：用户把同一话题的多条抖音视频链接贴入，AI 跨视频提取观点、识别共识与分歧、给出适用建议，输出层次分明、可溯源、可追问的「观点地图」，让用户从"越看越乱"变成"有能力判断"。

**核心场景**：用户刷了 5 条关于"鱼油到底有没有用"的视频，说法互相矛盾，离开时比进来更困惑。贴进这 5 条链接，AI 帮你看清：谁说了什么、哪里一致、哪里分歧、分歧根源、你该信谁。

**核心表达**：当同一个问题被不同视频说得互相矛盾时，观点地图帮你看清共识、分歧和适合自己的判断路径。

## 二、技术栈

- 前端：Next.js 14 App Router + TypeScript + Tailwind CSS；长图导出用 `html2canvas`
- 后端：Python + FastAPI，接口集中在单文件 `main.py`
- 视频信息：TikHub API
- 语音转文字：openai-whisper（本地，全局懒加载复用）
- 大模型：**DeepSeek（OpenAI 兼容接口，用 `openai` SDK 调用）**
- 项目结构：两个独立项目，分别放在 `frontend/` 与 `backend/`

### 4. ResultPage.tsx

```typescript
interface ResultPageProps {
```

顶部：话题名 + 定位文案「从刷到内容，到获得判断」+ 模式切换「卡片流 / 鸟瞰图」+「重新输入」按钮。

**卡片流模式**（核心，必须像「观点地图」不能像摘要）。用一个 `ref` 包住可导出区域，导出范围含区块 1–4，不默认导出追问区。

- 区块1 一句话结论：`analysis.one_line_summary`，作为整张地图总判断。
- 区块2 共识区：标题「共识区」，渲染 `consensus`，每条显示观点 + 可点击来源编号 `[1] [3]`（点击 `onViewRefs(id)`）。
- 区块3 分歧区：标题「分歧区」，渲染 `conflicts`，每个争议点显示标题 + 支持方 + 反对方 + 双方来源编号，支持/反对用不同颜色区分。
- 区块4 AI 建议：标题「AI 建议」，渲染 `recommendations`，每条显示条件 + 建议 + 来源编号；建议必须强调边界条件，不能写空泛鸡汤。
- 区块5 继续追问：标题「继续追问」，提示「只基于以上视频分析回答，不引入外部信息」；输入框 + 发送；对话历史列表；发送调用 `POST /api/followup`；维护 `history`；loading 禁用按钮；失败显示错误。

底部操作：「查看参考来源」按钮、「保存为长图」按钮。

**保存为长图**（html2canvas）：导出卡片流核心区域；文件名 `${topic}-观点地图.png`；导出中 loading；失败提示。若用户已追问过，在导出按钮旁提供 checkbox「导出时包含追问记录」。

**鸟瞰图模式**（纯 HTML + CSS + SVG，不引入 D3）：`relative` 容器高约 500px（桌面更高、移动自适应）；中心节点=共识区（显示「共识区」+ 共识条数）；周围节点=每个争议点（显示争议标题）；SVG 从中心连线到各争议点。节点位置：
```typescript
const total = analysis.conflicts.length
const angle = (2 * Math.PI * index) / total - Math.PI / 2
const x = 50 + 35 * Math.cos(angle)
const y = 50 + 35 * Math.sin(angle)
```
交互：点中心→切卡片流并滚到共识区；点争议节点→切卡片流并滚到对应分歧卡片；鸟瞰图下方展示选中争议点简要详情。

## 六、后端实现要求（backend/main.py）

FastAPI + CORS 全开。从 `.env` 读取（**全部由用户自行填写真实值**）：

```env
TIKHUB_TOKEN=
DEEPSEEK_API_KEY=
DEEPSEEK_BASE_URL=https://api.deepseek.com
DEEPSEEK_MODEL=deepseek-chat
```

> DeepSeek 用 `openai` SDK 调用（OpenAI 兼容）：
> ```python
> from openai import OpenAI
> client = OpenAI(api_key=os.getenv("DEEPSEEK_API_KEY"), base_url=os.getenv("DEEPSEEK_BASE_URL"))
> resp = client.chat.completions.create(
>     model=os.getenv("DEEPSEEK_MODEL", "deepseek-chat"),
>     messages=[{"role": "user", "content": prompt}],
>     max_tokens=4096,
>     temperature=0.3,
> )
> text = resp.choices[0].message.content
> ```
> 生成分析 JSON 时，若 base_url/model 支持，可加 `response_format={"type": "json_object"}` 提升 JSON 稳定性；同时仍保留解析容错+重试，不依赖此参数。

### 接口列表

- `GET /api/health` → `{ "ok": true }`
- `GET /api/preset/{preset_id}`：只允许 1/2/3，其他 404；读 `presets/{id}.json` 返回；读取失败 500。
- `POST /api/analyze`
- `POST /api/followup`

### POST /api/analyze

请求：`{ "links": ["https://www.douyin.com/video/xxx"], "topic": "鱼油到底有没有用" }`
返回：`Analysis`（五个顶层字段）。

**步骤1：并行提取所有视频**（`asyncio.gather`，同步操作用 `asyncio.to_thread` 包裹）

对每条链接：
1. 若含 `v.douyin.com`，用 `requests.get(..., allow_redirects=True)` 拿重定向后完整 URL。
2. 从完整 URL 提取视频 ID：优先 `/video/(\d+)`，兜底取末段纯数字。
3. 调 TikHub：
```python
headers = {"Authorization": f"Bearer {TIKHUB_TOKEN}"}
params = {"aweme_id": video_id}
resp = requests.get("https://api.tikhub.io/api/v1/douyin/web/fetch_one_video",
                    headers=headers, params=params, timeout=30)
detail = resp.json()["data"]["aweme_detail"]
title = detail.get("item_title") or detail.get("desc") or "未命名视频"
author = detail["author"]["nickname"]
audio_url = detail["music"]["play_url"]["uri"]
```
4. 下载 MP3 到临时文件。
5. Whisper 转写（模型全局懒加载复用）：
```python
model = get_whisper_model()
raw_text = model.transcribe(tmp_path, language="zh")["text"]
```
6. DeepSeek 清洗文字：
```text
将以下语音转录文本加上标点、去除明显口头语、纠正错别字。
要求：不改变原意，不总结，不删减任何观点，只输出修正后的文本。

{raw_text}
```

容错：某条失败则捕获异常、记录、跳过该条继续其余；**全部失败返回 502** 并提示用户使用预置话题。

**步骤2：DeepSeek 生成分析 JSON**

把所有视频清洗文字拼入 Prompt，要求只输出 JSON：

```text
你是一个专业的信息分析师。以下是 {N} 条关于「{topic}」的视频文字内容。
你的任务不是简单总结，而是把这些分散甚至互相冲突的视频，重构成用户能判断的观点地图。

【视频内容】
视频1（{author}，标题：{title}）：{clean_text}
视频2（{author}，标题：{title}）：{clean_text}
...

请完成：
1. 提取每条视频的核心主张。
2. 找出多数视频都认同的共识。
3. 找出存在对立或适用条件不同的分歧。
4. 根据不同用户情境给出可执行建议。
5. 所有观点必须标注来源编号，例如 sources 使用 [1, 3]。

严格按以下 JSON 格式输出，不输出任何其他内容：
{
  "one_line_summary": "一句话总结整体判断，必须体现共识、分歧或适用边界",
  "consensus": [ { "point": "共识观点", "sources": [1, 3] } ],
  "conflicts": [ {
      "topic": "争议点标题",
      "pro": { "argument": "支持方观点和理由", "sources": [2] },
      "con": { "argument": "反对方观点和理由", "sources": [3] }
  } ],
  "recommendations": [ {
      "condition": "如果你是 XX 情况",
      "advice": "具体可执行建议，必须有明确边界条件，不能泛泛而谈",
      "sources": [1, 2]
  } ],
  "references": [ {
      "id": 1, "author": "作者名", "title": "视频标题",
      "claim": "该视频核心主张一句话", "url": "原链接"
  } ]
}
```

JSON 解析：首次失败时调 DeepSeek 重试一次（Prompt 要求修复为合法 JSON）；再失败返回 500；返回前校验包含五个顶层字段。

### POST /api/followup

请求：`{ "analysis": {}, "question": "我适合吃鱼油吗？", "history": [{"role":"user","content":"..."},{"role":"assistant","content":"..."}] }`
返回：`{ "answer": "..." }`

Prompt：
```text
已分析内容：
{json.dumps(analysis, ensure_ascii=False)}

对话历史：
{history}

用户问题：
{question}

请只基于已分析内容回答。
如果引用视频，必须使用来源编号，例如 [1]、[2]。
如果已分析内容没有涉及，直接回复：被分析的视频中没有涉及这个问题。
不要补充任何外部知识。
```
