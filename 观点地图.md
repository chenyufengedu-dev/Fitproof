## 一、产品定义

### 一句话
用户把同一话题的多条抖音视频链接贴入，AI 跨视频提取观点、识别共识与分歧，输出层次分明的"观点地图"，让用户从"越看越乱"变成"有能力判断"。

### 核心场景
用户刷了5条关于"鱼油到底有没有用"的视频，每条说法不同甚至相互矛盾。离开抖音时比进来时更困惑。现在把这5条链接贴进来，AI 帮你看清：谁说了什么、哪里一致、哪里分歧、分歧根源是什么、你该信谁。

### 为什么没有替代品
| 现有工具 | 能做什么 | 做不到什么 |
|---|---|---|
| 豆包 | 总结单条视频 | 无法跨多条视频交叉对比 |
| 百度搜索 | 找新内容 | 无法整合你已经看过的内容 |
| 各类App | 垂直领域工具 | 无法处理用户自选的视频内容 |

## 三、用户体验流程

### 页面一：输入页
- 输入框：粘贴 2–5 条抖音视频链接（每行一条）
- 可选填写话题名称（如"鱼油是否有用"）
- "开始分析"按钮
- 下方两个预置话题快捷按钮（评委无需自己找链接，点击了会自动将预置好的内容放到输入框中，填写好输入框）

### 页面二：加载页
四步进度依次亮起，每步完成有勾选动画：
1. 正在提取视频内容…
2. 正在识别核心观点… → 完成时显示"发现 N 个争议点"
3. 正在交叉对比…
4. 生成观点地图…
（优化的时候可以加更好的加载动画）

### 页面三：观点地图（核心页面）

顶部切换：`卡片流` | `鸟瞰图`

**卡片流模式（默认，五层从上往下  适用于手机端）**

卡片流模式：出现类似于探探 app 的卡片模式，左滑是进入下一个卡片，右滑出现下一张卡片，第一张的话左滑没用，最后一张卡片如果右滑则进入下一个内容


```
┌─────────────────────────────────────┐
│  一句话结论                          │  ← AI对这批视频的整体判断
└─────────────────────────────────────┘

┌─────────────────────────────────────┐
│  ✅ 共识区                           │  ← 所有/多数视频都认同的部分
│  · 观点A  [来源：视频1, 视频2, 视频3] │
│  · 观点B  [来源：视频1, 视频4]       │
└─────────────────────────────────────┘

┌─────────────────────────────────────┐
│  ⚡ 分歧区                           │  ← 存在对立的部分
│  争议点1：XXX                        │
│    支持方：论据 [来源：视频2]         │
│    反对方：论据 [来源：视频3, 视频5]  │
│  争议点2：YYY                        │
│    ...                              │
└─────────────────────────────────────┘

┌─────────────────────────────────────┐
│  🤖 AI 建议                         │  ← 根据不同情境给出针对性判断
│  如果你是XX情况 → 建议YY             │
│  如果你是XX情况 → 建议YY             │
└─────────────────────────────────────┘

┌─────────────────────────────────────┐
│  💬 继续追问                         │  ← 基于已分析内容回答，不搜索外部
│  [输入框] 对这个话题还有疑问？        │
└─────────────────────────────────────┘
```

**鸟瞰图模式（切换后展示 适用于电脑端）**

- 中心节点：共识区
- 环绕节点：每个争议点（简述双方立场）
- 点击任意节点跳回卡片流对应位置
- 纯 CSS + HTML 实现，不引入 D3.js

每个页面中的内容如果是从网上找来的要标上类似于 SCI 论文的参考文献序号 “[1]” 超链接，点击会进入到页面四：参考来源对应的参考文献处。
### 页面四：参考来源
列出所有被分析视频：标题 · 博主名 · 核心主张 · 原链接  
格式参考论文参考文献，强化专业感与可溯源性。
如果是通过点击观点地图核心页面来到页面 4，应该出现一个返回的按钮回到原来的页面。如果是顺其自然来到页面四：参考来源 则不用出现返回按钮。

### 导出
页面底部固定按钮"📥 保存为长图"  
使用 html2canvas 将卡片流一到四层渲染为长图，可保存或分享。
如果可以选择的话，可以在输出中选择是否一起输出对话内容，如果是用户进行了追问，如果没有对话过就不用有这个选项。

### 核心 Prompt（一次调用，输出完整 JSON）

```
你是一个专业的信息分析师。用户提供了 {N} 条关于"{话题}"的视频内容。

【视频内容】
视频1（{博主名}）：{转录文本}
视频2（{博主名}）：{转录文本}
...

请完成以下分析并严格按 JSON 格式输出，不要输出任何其他内容：

1. 逐个视频提取：核心主张、关键论据、引用的数据或证据
2. 识别共识：所有或多数视频都认同的观点
3. 识别分歧：存在对立的争议点，列出双方论据和来源
4. 综合判断：根据不同用户情况给出针对性建议

输出 JSON 格式：
{
  "one_line_summary": "一句话总结核心分歧和判断",
  "consensus": [
    {"point": "共识观点", "agreed_by": ["视频1", "视频3"]}
  ],
  "conflicts": [
    {
      "topic": "争议焦点标题",
      "positions": [
        {"side": "支持方", "arguments": ["论据1", "论据2"], "held_by": ["视频1"]},
        {"side": "反对方", "arguments": ["论据1"], "held_by": ["视频2", "视频4"]}
      ]
    }
  ],
  "recommendation": {
    "summary": "综合判断一句话",
    "by_scenario": [
      {"condition": "如果你是XX情况", "advice": "建议YY"}
    ]
  },
  "references": [
    {"video_id": "视频1", "creator": "博主名", "core_claim": "核心主张一句话", "link": "原链接"}
  ]
}
```

### 追问 Prompt

```
你之前已分析了以下内容：
{完整分析结果 JSON}

用户追问："{问题}"

只基于以上分析结果回答。如果涉及，详细回答并标注来源视频。
如果不涉及，回答："被分析的视频中没有涉及这个问题。"
不要补充外部知识。
```

### 接口定义（前后端约定）

```
POST /api/analyze
Request:  { "links": ["url1", "url2"], "topic": "话题名称" }
Response: { 分析结果 JSON }

POST /api/followup  
Request:  { "analysis": {分析结果}, "question": "用户问题" }
Response: { "answer": "回答文本" }

GET /api/preset/{id}
Response: { 预置话题的分析结果 JSON }
```
