## 一、后端 / 分析管线

### P0 — 2.1b：ASR 换批量接口 + 音频URL直传（已出提示词）
- dashscope 分支从实时模型 `paraformer-realtime-v2` 换成**录音文件识别(批量, paraformer-v2)**：几秒转完 + 完整句级时间戳。
- 直接把 TikHub 的抖音音频URL喂给它，免下载免转码；失败回退下载/本地Whisper。
- 目标：单视频十几秒出结果。

### P1 — 全文块兜底层（三级检索的第2级）
- 每份文档全文切成 300-500字带重叠小块 → bge向量 → 单独 chunks 索引（复用 evidence_store）。
- 核验检索三级：①结论库(1472条) → ②全文块兜底 → ③AI常识(诚实降级)。级别间用相关度阈值切换。
- 解决"结论抽取有损、别丢信息"，同时保留结论库作为主路径身份。

### P2 — 画面理解（视觉模型，修正后的设计）
- **不用场景检测**（它要下整段视频，慢）。改：**定时采样(每~5s)用HTTP Range单帧抓取(不下整段) → 感知哈希去重 → 硬封顶N帧(~6-8) → Qwen-VL-Flash并行解读("无关回无"过滤) → 帧描述+时间戳汇总喂核验**。
- 视觉模型先用 Flash 看效果，不够再针对性升级；只处理去重封顶后的少数帧。

### P2 — 提取管线并行化
- 音频线(ASR) ‖ 画面线(采样+视觉) 并行，join 后再拆主张。依赖画面选帧不再靠转写(采样法天然独立)。

## 二、前端

### P1 — 可信度"溯源式"展示（不造百分比）
- 不发明置信度算法（重蹈老系统正则的坑）。直接展示：判定来自**哪一级 + 机构 + 推荐强度**。
  - 命中结论→"可信度高·依据《X指南》指南推荐"；命中全文块→"可信度中·来自X指南原文段落"；AI兜底→"可信度低·未收录依据,AI常识判断"。
- 技术极简：模型已输出 confidence/strength，只需再显示来源级别+机构名。

### P2/加分 — 3.4 三Tab骨架
- 核验(核心已成) / 社区(预置卡片演示版) / 我的(核验历史回看+核验成就)。
- 核验成就走"求真"向(已核验N条/避开N个误导)，只在"我的"展示层，不娱乐化。
