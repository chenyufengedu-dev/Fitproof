## 二、风格规范（最重要，决定统一性）

现有 36 张的问题是**风格不统一**：鸡蛋是扁平简约的、疫苗是写实精细的（玻璃质感+注射器刻度）、孕妇是卡通人物、通用图是拟物按钮。四种画风混在一排里很乱。

**新图必须锁定一种风格。推荐这一种：**

```
半拟真 3D 渲染图标，圆润几何体块，柔和的顶部打光，
轻微的环境光遮蔽，无描边，无投影，无文字，无背景装饰
```

### 硬性要求

| 项 | 要求 | 为什么 |
|---|---|---|
| **背景** | **纯透明**（PNG alpha 通道） | 要放进不同底色的圆形容器 |
| **画布** | 正方形，≥1000×1000 | 留足缩放余量 |
| **构图** | 单个主体，居中，**占画布 60~75%** | 现有 36 张占比从 24% 到 38% 不等，放进同一个圆底里大小忽大忽小 |
| **视角** | **全部统一**（建议 3/4 俯视） | 有的正视有的侧视，排在一起会乱 |
| **投影** | **不要** | 小尺寸下投影糊成一团脏东西 |
| **描边** | **不要** | 46px 显示时描边会吃掉内部细节 |
| **文字** | **绝对不要** | 缩到 34px 完全看不清，只是噪点 |

### 配色

主色跟随产品：**医疗青绿 `#20CDB6` / 深青 `#0B6E63`**

- 物体**本色优先**（鸡蛋就是黄白色、番茄就是红色），不要强行全染成青色
- 但**中性物件**（药盒、报告单、仪器）用青绿系
- **避免**：高饱和荧光色、纯黑、大面积灰

## 三、提示词模板

直接套用，把 `[物体]` 换掉：

```
一个 [物体] 的 3D 图标，半拟真风格，圆润几何造型，
柔和顶部打光，3/4 俯视角度，居中构图占画面约 70%，
纯白背景，无阴影，无描边，无文字，
配色干净以青绿色 #20CDB6 为辅助色，
适合作为健康类 App 的小图标，1024x1024
```

生成后**抠成透明底**（remove.bg / Photoshop / 在线抠图都行），存成 **PNG**。

## 五、交付方式

1. 所有 PNG（透明底）放进一个文件夹，比如 `frontend/public/materials/new-icons/`
2. **文件名可以随意**（`ChatGPT Image xxx.png` 也行），但**必须给一份顺序清单**说明第几张是什么，例如：
   ```
   1. seafood  鱼虾海鲜
   2. nut      坚果
   3. fruit    水果拼盘
   ```
3. 交给主 AI，剩下的处理是自动的

## 六、后续处理流程（主 AI 执行，这里写明供参考）

项目里已有现成脚本 `frontend/scripts/process-icons.py`：

```bash
cd D:\PointMap\frontend
python scripts/process-icons.py public/materials/new-icons public/claim-icons-v3
```

脚本会自动：
1. 裁到 alpha 边界（消除不同图的留白差异 → **解决大小不一**）
2. 长边等比缩到 160px，四周补 6% 留白
3. 转 webp，quality=88

**实测效果：20.0 MB → 190 KB，压缩 99%，每张 4-5KB。**

然后：
- 按清单重命名
- 拷进 `frontend/public/claim-icons/`
- 把新名字加进 `backend/main.py` 的 `CLAIM_ICONS` 白名单（**不加的话模型不会用到这些新图**）
- 同步更新 `main.py` 提示词里的 icon 可选清单

## 七、验收标准

新图和现有 36 张放在一起，应该看不出是两批做的：

- [ ] 透明背景，没有白色/灰色残底
- [ ] 主体占画布 60~75%，各张之间差异不超过 10%
- [ ] 视角统一
- [ ] 无描边、无投影、无文字
- [ ] 缩到 **34px** 看仍能认出是什么（这是实际显示尺寸，务必自己缩小验一遍）
- [ ] 处理后单张 ≤ 8KB
