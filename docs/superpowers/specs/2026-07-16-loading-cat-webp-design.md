# 加载页核验猫 WebP 动画设计

## 目标

将加载页现有的 `frontend/public/brand/cat-checking.png` 重制为透明背景的动画 WebP 吉祥物。动画保持原有小猫拿放大镜查看平板资料的线条风形象，用于表达 FitProof 正在核验证据。

## 视觉与动效

- 保留黑色手绘轮廓、猫咪、平板与放大镜的核心姿态；不引入渐变、紫色或卡片式装饰。
- 动画时长约三秒并循环：放大镜在平板资料区域平稳扫过；镜片内出现浅青色的短扫描线；扫描完成后的停顿中，猫咪进行一次短暂眨眼。
- 输出为透明背景 animated WebP，以适配加载页的浅青色信息卡和任何未来背景。
- 资源以较大的源尺寸导出，加载页中仍以现有约 56px 的视觉尺寸展示，保证高密度屏幕清晰度。

## 前端集成

- 新资源命名为 `frontend/public/brand/cat-checking.webp`。
- `LoadingPage` 的普通 `<img>` 优先指向 WebP，不新增 Lottie 播放器、运行时依赖或额外状态管理。
- 现有 `fitproof-cat-float` 只保留轻微浮动，幅度会相应收敛，避免和动画本身的放大镜动作竞争。
- 当 WebP 无法加载时，图片元素切换回既有的 `cat-checking.png`，以保证加载页始终有可用吉祥物。

## 验证

1. 检查 WebP 文件能在浏览器内循环播放，透明背景、放大镜扫描和眨眼节奏均清晰自然。
2. 在加载页中检查 56px 展示位下不裁切、不模糊、不遮挡主题文字。
3. 模拟动画资源加载失败，确认自动显示原 PNG。
4. 运行 `npx tsc --noEmit`，确认前端类型检查通过。

## 非目标

- 不改动加载流程、进度文案或其他页面。
- 不引入 Lottie、GIF 或新的第三方依赖。
