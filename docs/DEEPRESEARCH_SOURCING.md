# 深度搜索「找源」提示词（给用户用）

> 用途：用 GPT / Gemini 的深度搜索，**只找真实权威原件的下载线索**，不要它的结论。
> 找到后你顺着 URL 下载原件（指南/综述/官方文件）放进 `backend/evidence/raw/`，再交给拆条 AI。
> **深度搜索的报告正文一律不进库**——它只是"帮我找到源头在哪"的检索工具。

## 用法

一个领域跑一次（省额度）。把下面提示词里 `【领域】` 换成你要找的领域，整段发给深度搜索。
GPT 和 Gemini 可分工跑不同领域，不重复。

## 提示词（复制这段）

```
你是一名循证医学资料员。我在为一个健康信息核验工具收集【权威原始文档】，
需要你帮我"找源"，而不是给结论。

【领域】孕期与哺乳期营养

请围绕这个领域中普通人最常搜、最容易被短视频误导的常见说法（列 12~15 个），
为每个说法找出【可以下载或在线查阅的真实权威原始文档】。

【只找这几类来源，按可信度从高到低】
1. Cochrane 系统综述 / 权威期刊的 Meta 分析（给 PubMed 或 Cochrane 官方页面链接）
2. 官方膳食/临床指南、权威机构立场声明（中国营养学会、国家卫健委、WHO、ACOG 等官网）
3. 权威医疗机构循证科普原页（Mayo Clinic、默沙东手册、UpToDate 公众版、WHO Fact Sheet）

【严格要求 —— 这是找源任务，不是总结任务】
- 你的输出重点是【真实、可点击、指向官方来源的 URL】，不是结论内容。
- 每条只需：来源名称 + 发布机构 + 类型 + 年份 + 官方链接 + 它覆盖了哪个常见说法。
- 优先给【能直接下载 PDF 或官方全文页】的链接，而不是二手转载、新闻或博客。
- 严禁编造链接、DOI、机构名。找不到可靠官方来源的说法，直接写"未找到可靠来源"。
- 中文官方来源优先；权威英文来源可给，标注"英文"。
- 不要长篇论述该说法的医学结论，我只要来源清单。

【输出格式】一张表：
| 常见说法 | 来源名称 | 发布机构 | 类型(综述/指南/科普) | 年份 | 官方下载/查阅链接 | 语言 |
```

## 领域清单（按优先级，一个跑一次）

**必做（演示要用，先跑这 4 个）**
1. 孕期与哺乳期营养
2. 婴幼儿与儿童喂养/健康
3. 运动健身（含空腹有氧、增肌减脂常见说法）
4. 日常膳食与常见食物谣言（鸡蛋/胆固醇、鱼油、保健品、酱油、隔夜菜等）

**有余力再跑**
5. 慢病常识（三高、血糖、血脂）
6. 睡眠与作息
7. 常见用药/补剂误区

## 拿到结果后你要做的（关键）

1. **抽查链接**：随手点开几个，确认能打开、是官方来源（尤其 pubmed / cochrane / who / 政府和学会官网）。深度搜索最爱在这里编链接，编的一律弃用。
2. **下载原件**：能下 PDF 的下 PDF；只有网页的用浏览器"打印→另存为 PDF"。存进 `backend/evidence/raw/`，文件名用拼音/英文短名。
3. **登记**：在 `docs/EVIDENCE_COLLECTION.md` 第四节表格加一行（含官方链接）。
4. 之后交给拆条 AI（按 `docs/EVIDENCE_INGEST_SPEC.md`）。

## 注意
- 英文原件也可以收（拆条脚本能处理），但优先中文，评委看中文官方来源最直观。
- 单篇原始研究（普通 RCT、观察性研究）**不要收**——只收系统综述/Meta 分析/指南/权威科普这类"已消化过的高层级结论"。
- 付费墙打不开全文的，收它的官方摘要页也行，能溯源即可。
