# 证据库搭建 · 团队分工（3 人并行）

> 目标：一周内建成核验证据库的第一批（约 300~500 条）。
> 原则：**按领域包干，每人端到端负责自己的领域**，互不干扰、不用互相等。

## 一、三条必读（所有人）

1. 找源用 `docs/DEEPRESEARCH_SOURCING.md` 的提示词（深度搜索只找真实原件链接，**它的结论不要**）。
2. 拆条规范看 `docs/EVIDENCE_INGEST_SPEC.md`（每条怎么拆、什么格式）。
3. **红线**：只收系统综述/Meta分析/指南/权威科普，不收单篇普通论文；深度搜索报告本身**绝不进库**，只用它找来源。

## 二、分工（按领域包干，避免撞车）

| 负责人 | 包干领域 | 备注 |
|---|---|---|
| **组员 A** | ①孕期与哺乳期营养　②婴幼儿与儿童喂养 | 母婴线，演示重点 |
| **组员 B** | ③运动健身（含空腹有氧）　④日常食物谣言（鸡蛋/鱼油/保健品等） | 老本行 + 谣言高发区 |
| **你（负责人）** | 统筹 + 验收 + 4 个演示话题的证据把关 | 见第五节 |

有余力再扩：⑤慢病常识 ⑥睡眠 ⑦用药误区——等前 4 个领域完成后再分。

## 三、每人的标准流程（对自己每个领域走一遍）

1. **找源**：用深度搜索提示词跑该领域 → 得到来源清单。
2. **验链接**：逐个点开，确认能打开、是官方来源（政府/学会/WHO/Cochrane/PubMed 官网）。**编的、打不开的、二手转载的一律弃用。**
3. **下载原件**：能下 PDF 下 PDF，只有网页的用浏览器"打印→另存 PDF"。存进 `backend/evidence/raw/`。
4. **命名规则（防覆盖）**：文件名 = `领域前缀-文档短名.pdf`，前缀用：孕期=yq、婴幼=yy、运动=yd、膳食谣言=ss、慢病=mb。例：`yq-yunqi-shanshi-zhinan.pdf`。
5. **登记**：填登记表（见第四节）。
6. **拆条**：把该文档 + `EVIDENCE_INGEST_SPEC.md` 交给一个 AI（或自己跑 `ingest_evidence.py`），产出到 `backend/evidence/entries/`，文件名同样带领域前缀。
7. **自查**：抽 10~20% 条目回原文核对，没编造、没夸大。

## 四、登记表怎么填（防冲突）

**不要三个人同时编辑同一个文件**，否则会互相覆盖。二选一：
- 推荐：建一个**共享在线表格**（腾讯文档/飞书多维表），列同 `EVIDENCE_COLLECTION.md` 第四节，三人各填自己的行。
- 或者：每人维护自己的登记文件 `docs/registry-A.md` / `registry-B.md`，最后由负责人合并。

每行必填：文档名 / 发布机构 / 年份 / **官方链接** / 本地文件名 / 领域 / 收集人 / 状态。

## 五、你（负责人）的额外职责

1. **4 个演示话题的证据要最硬**：孕妇吃蛋黄、空腹有氧、鱼油智商税、+1 个宝妈向。这几个话题命中的证据条目你亲自复核，演示时评委最可能追问这几条。
2. **每天收口一次**：让两位组员报"今天下了几份、拆了多少条"，你抽查几条链接和条目。
3. **汇总交付**：所有 `entries/*.json` 攒到一起，通知主工程师（AI）做去重 + 建检索索引 + 入库。

## 六、质量红线（写在最前面天天提醒）

- ✅ 每条 claim 能在原件里找到依据　✅ 每个 URL 真实可打开的官方链接
- ❌ 不收单篇普通论文　❌ 深度搜索/AI 生成内容不进库　❌ 不编造数字、机构、结论
- **宁可少而真，不要多而假。** 我们整个产品就靠"来源真实可溯源"立足，这一关破了全盘皆输。

## 七、进度目标（参考）

- Day 1-2：每人本领域找源 + 下载 8~12 份原件
- Day 3-4：拆条 + 自查，产出各领域 entries
- Day 5：负责人汇总验收，交主工程师入库
- 首批目标：4 领域合计 300~500 条
